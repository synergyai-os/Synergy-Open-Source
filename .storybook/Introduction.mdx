## ğŸ¨ Key Concepts

### Design Tokens

Semantic variables that adapt to light/dark mode automatically.

**Example:**

- `--spacing-button-x` (not hardcoded `24px`)
- `--color-accent-primary` (not hardcoded `#3b82f6`)

**Why?** Change once in `app.css`, updates everywhere. Dark mode automatic.

**âš ï¸ CRITICAL**: Hardcoded Tailwind values are blocked by ESLint. Always use semantic tokens.

### âš›ï¸ Atomic Design

We organize components using atomic design methodology:

- **Atoms**: Single elements (Button, Input, Card, Badge)
- **Molecules**: 2-3 atoms composed (FormField = Label + Input + Error)
- **Organisms**: Complex sections (Header, Sidebar, Dialog)

**Why?** Clear hierarchy, reusability, composition over complexity.

### ğŸ”„ The Cascade

When you change `--spacing-button-x` in tokens:

1. Updates `px-button-x` utility
2. Updates Button component
3. Updates all pages using Button

**No manual updates needed!**
