## Journey Format

Each journey follows this structure:

```markdown
# Journey Name

> One-sentence summary

## User Goal

What the user wants to accomplish and why.

## Preconditions

What must be true before starting (permissions, data, setup).

## Steps

### Step 1: [Action]

**What**: User does X
**Where**: Page/component name
**Screenshot**: [Image]
**Notes**: Edge cases, tips

### Step 2: [Action]

...

## Success Criteria

How user knows they've succeeded.

## Edge Cases

- What if Y happens?
- Alternative path Z

## Related Journeys

- [Other Journey](./other-journey.md)
- [Component](../component-library/component.md)
```
